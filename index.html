2. Rendering Pipeline Three-layer system

The Dither Machine uses a three-layer rendering system so that background dither, active gameplay, and optional effects all remain cleanly separated and composable.

2.1 Background Dither Layer (Type A Animated Dither)

Full-screen canvas positioned behind all game/UI elements.

  • Animated dithering pattern with smooth, flowing gradient movement (noise-based drift; no linear sliding).
  • Adjustable parameters:
    • Dot size, density, and blend softness
    • Contrast and color palettes
    • Movement speed and motion style
  • No visual seams or tiling lines ‚Äî pattern must internally wrap and noise-blend.
  • Multi-resolution support that scales cleanly from 1080p up to 4K.
2.2 Scene Layer (Normal Game Rendering)
  • Foreground game and UI elements are rendered normally and remain crisp and non-dithered by default.
  • Separation ensures icons, text, and UI remain readable (similar to Discord‚Äôs clean icons over dithered backgrounds).
  • Optional ‚Äúdepth-aware‚Äù dithering:
    • Foreground can smoothly fade into background dither as elements move further away.
    • Depth simulation can be based on Z-index, parallax layers, or game-defined depth values.
2.3 Optional Effect Layers
  • Light rays, bloom, shadows, or gradient fields that can apply localized dithering distortions.
  • Animated masks (radial bursts, wave pulses, etc.) that interact with the underlying dither pattern.

3. Dither Modes & Algorithms Switchable & layerable

The Dither Machine supports multiple dithering modes that can be swapped at runtime or layered to achieve different visual styles.

3.1 Ordered Dithering
  • Support for Bayer matrices:
    • 2√ó2, 4√ó4, and 8√ó8 threshold maps.
  • Pixel-precise control over:
    • Threshold map scaling
    • Pattern rotation
    • Pattern scrolling / offset
3.2 Error Diffusion Dithering
  • Algorithms:
    • Floyd‚ÄìSteinberg
    • JJJN
    • Atkinson
    • Stucki
  • Real-time, GPU-accelerated diffusion computation (implemented via shaders rather than CPU loops).
3.3 Hybrid Dither Mode
  • Supports combining:
    • Ordered + noise map
    • Ordered + animated gradient
    • Diffusion + pixel cell clustering
3.4 Animated Gradient Dither (Type A)

This is how you get the Discord-style smooth, flowing motion:

  • Multilayer animated gradient mapped into a dither threshold.
  • Movement characteristics:
    • Slow (no harsh or jittery motion).
    • Organic noise-driven drift (Perlin/Simplex noise field).
    • Curved, drifting flow rather than straight linear scrolling.
3.5 Posterization Controls
  • Color banding / bit-depth control.
  • 2‚Äì32 color levels.
  • Ability to lock the color palette to:
    • Monochrome
    • Duotone
    • Neon (soft, non-clashing)
    • Pastel (soft, harmonious)
    • Retro pixel palettes

6. Integration Features Game / UI ready

6.1 Foreground Priority System
  • Guarantees UI is always readable on top of the dither.
  • Smart luminance detection:
    • Dither stays darker behind light UI.
    • Dither stays lighter behind dark UI.
  • Automatic 10‚Äì30% contrast buffer around icon and text edges.
6.2 Parallax Dither Motion
  • Background dither gently shifts based on:
    • Mouse movement.
    • Player movement.
    • Camera movement.
  • Effect is subtle and optional, but adds a premium, ‚Äúalive‚Äù feel to the background.
6.3 Blend-Over Gameplay

Engine can target specific render paths:

  • Entire game scene.
  • Only background.
  • Only shadows.
  • Only lighting.

Each can be toggled independently.

7. Performance Architecture Keep it fast

7.1 GPU Acceleration
  • Use WebGL2 or WebGPU for shading pipeline.
  • OffscreenCanvas for multi-threaded rendering where supported.
  • Shader-based dithering and threshold evaluation (avoid heavy CPU loops).
7.2 Dynamic Resolution Scaling
  • If framerate drops:
    • Reduce dither canvas resolution first.
    • Keep primary game/foreground layers crisp.
7.3 Zero-Seam Rendering
  • Procedural patterns tile at the shader level.
  • No artifact lines or edge seams as the pattern scrolls/moves.

9. Special Effects (Optional) Bonus features

9.1 Ripple Distortion Dither
  • Dither pattern warps like water ripples, driven by noise or specific events.
9.2 Light-Reactive Dithering
  • Bright areas have finer, more subtle dithering.
  • Dark areas use chunkier, more pronounced dither blocks.
9.3 Audio-Reactive Dither
  • Dots pulse or shift based on music bass or sound amplitude.
9.4 Particle-Dither Interaction
  • Particles temporarily disturb or displace the dither field behind them.
